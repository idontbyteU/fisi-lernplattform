🧭1 · Ablauf eines Softwaretests

Ein Softwaretest läuft in sechs Phasen ab. Quer über alle Phasen liegt die Steuerung — sie plant, überwacht und justiert den gesamten Testprozess.

  • Testplanung — Ziele festlegen
  • Testentwurf (Analyse & Design) — was wird wie getestet
  • Testvorbereitung — konkrete Testszenarien und Testfälle festlegen
  • Testdurchführung — die Testfälle ausführen
  • Testauswertung — Testprotokolle erstellen
  • Testabschluss
PlanungEntwurfVorbereitungDurchführungAuswertungAbschluss

Steuerung ist ein Querschnitt über alle sechs Phasen.

📌
Merkkette: Planen → Entwerfen → Vorbereiten → Durchführen → Auswerten → Abschließen. Die Steuerung läuft über alle Phasen mit.

📐2 · Softwarequalitätskriterien (ISO 9126)

Die Norm beschreibt sechs Hauptmerkmale für Softwarequalität, jeweils mit Untermerkmalen. In allen Gruppen kommt zusätzlich die Konformität (Normentreue) hinzu.

Funktionalität

Korrektheit/Richtigkeit, Vollständigkeit, Angemessenheit, Interoperabilität, Ordnungsmäßigkeit, Sicherheit, Konformität.

Zuverlässigkeit / Stabilität

Reife, Fehlertoleranz, Wiederherstellbarkeit, Konformität.

Benutzbarkeit

Verständlichkeit, Erlernbarkeit, Bedienbarkeit (UI, Benutzer-Doku), Konformität.

Effizienz

Zeitverhalten, Verbrauchsverhalten (Codeoptimierung), Konformität.

Änderbarkeit / Wartbarkeit

Analysierbarkeit, Modifizierbarkeit, Stabilität, Prüfbarkeit (Quellcode vorhanden, kommentiert), Konformität.

Übertragbarkeit

Anpassbarkeit, Installierbarkeit, Austauschbarkeit, Konformität.

📌
Besonders Funktionalität und Zuverlässigkeit lassen sich gut testen.
ℹ️
Hinweis: ISO 9126 wurde 2011 durch ISO/IEC 25010 abgelöst. Die Kriterien sind inhaltlich sehr ähnlich; in IHK-Aufgaben taucht oft noch 9126 auf.

🔬3 · Testverfahren — wie wird getestet?

Statisch
Code läuft nicht. Schreibtischtest (allein), Code Review (im Team), Code-Analyse (oft automatisch).
Dynamisch
Code läuft. Blackbox (ohne Codekenntnis) / Whitebox (mit Codekenntnis).
📌
Statisch = Code lesen/prüfen. Dynamisch = Code ausführen.

🔺4 · Teststufen — in welchem Umfang? (Testpyramide)

Von unten (viele) nach oben (wenige):

  • Unit-Test — einzelne Funktion/Methode/Modul/Klasse — Basis, viele Tests
  • Integrationstest — Zusammenspiel der Units
  • Acceptance- / Abnahmetest — ganze Anwendung inkl. UI im realen Szenario — Spitze, wenige Tests
📌
Testpyramide: viele kleine Unit-Tests unten, wenige große Abnahmetests oben.

🧩5 · Unit-Tests (Detail)

  • Wichtigster Testbereich: die meisten funktionalen Fehler nach dem Coding werden hier gefunden.
  • Testet einzelne Einheiten (Funktion/Methode/Modul/Klasse, je nach Sprache).
  • Ziel: vollständige Codeüberdeckung (100 % des Codes soll ausgeführt und getestet werden).
  • Werden häufig automatisiert.

🔁6 · Testgetriebene Entwicklung (TDD)

  • Die Ergebnisse der Tests treiben die weitere Entwicklung.
  • Typisch: Verteilen von Alpha-, Beta-Versionen und Release Candidates außerhalb des Entwicklungsteams.
  • Meist in agilen Vorgehensmodellen (z. B. Scrum) üblich.
ℹ️
Kern von TDD im engeren Sinn: Test zuerst schreiben, dann Code. Zyklus rot (Test schlägt fehl) → grün (Code bis Test besteht) → refactor (aufräumen).

📚7 · Dokumentation

Benutzerdoku
Für Anwender/Administratoren: Benutzerhandbuch, Installationsanleitung, Schulungsunterlagen, Werbematerial.
Entwicklerdoku
Für Projektleiter, Entwickler, Tester: beschreibt Entwicklungsprozess sowie Aufbau und Funktionsweise des Systems; enthält den dokumentierten Quellcode und die Testdokumentation inkl. Testprotokolle.

🐍8 · Unit-Tests in Python: assert

  • assert ist ein Keyword, keine Funktion — also ohne Klammern aufrufen.
  • Bedingung True → Code läuft weiter (Testfall OK). Bedingung False → AssertionError.
  • Optionaler Fehlertext nach Komma.
  • Ein fehlgeschlagener assert beendet das Programm; mit try/except abfangbar.
def stellenzahl(x: int):            # zu testende Funktion
    return len(str(int(x)))         # Länge des Arguments (als int)

def test_stellenzahl():             # der eigentliche Modultest
    assert stellenzahl(100) == 3    # erwartet 3
    assert stellenzahl(50.78) == 2  # int(50.78)=50 -> "50" -> 2
    assert stellenzahl(-1000) == 4  # schlägt fehl!

test_stellenzahl()
⚠️
Klammer-Falle: assert ist ein Keyword. Richtig: assert funktion(x)==1 — Falsch: assert(funktion(x)==1). Bei -1000 schlägt der Test fehl, weil "-1000" fünf Zeichen hat (das Minus zählt mit), nicht vier.

Fehlertext-Beispiel (eigene Zeile):

assert 2+2 == 5, "Houston, wir haben ein Problem"

✅9 · pytest — das Python-Test-Tool

  • Nicht in Python enthalten: Installation mit pip3 install pytest
  • Aufruf im Projektordner: pytest meinprog.py
  • Führt alle Funktionen aus, deren Name mit test_ beginnt.
  • Zeigt fehlgeschlagene und erfolgreiche Tests an.
📌
Nur Funktionen mit Präfix test_ werden von pytest automatisch ausgeführt.
