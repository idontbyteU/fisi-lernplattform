📦 Was ist JSON?

JSON (JavaScript Object Notation) ist ein leichtgewichtiges, textbasiertes Datenformat, um strukturierte Daten zu speichern und zwischen Programmen auszutauschen. Es ist für Menschen gut lesbar und für Maschinen leicht zu verarbeiten. Der Name kommt aus JavaScript – JSON ist aber sprachunabhängig und wird von praktisch jeder Programmiersprache verstanden.

🎯
Kernidee: JSON ist einfach Text, der eine Struktur aus Objekten (Schlüssel→Wert) und Arrays (Listen) beschreibt. Ein Programm wandelt diesen Text in seine eigenen Datenstrukturen um – und wieder zurück.
Wofür wird JSON benutzt?

Web-APIs

REST-Schnittstellen senden und empfangen Daten fast immer als JSON (z. B. die Antwort eines Wetter- oder Login-Dienstes).

Konfiguration

Einstellungsdateien wie package.json, manifest.json oder VS-Code-Settings sind JSON.

Speichern & Austausch

Daten dauerhaft ablegen oder zwischen Systemen übertragen – kompakt, eindeutig, plattformneutral.

Browser-Speicher

localStorage speichert nur Text – Objekte werden als JSON abgelegt (genau wie in dieser Lernplattform).

NoSQL-Datenbanken

Dokumenten-Datenbanken (z. B. MongoDB) speichern Datensätze in JSON-ähnlicher Form.

Logs & Messaging

Strukturierte Logdaten und Nachrichten zwischen Diensten werden häufig als JSON-Zeilen übertragen.

🧱 Die zwei Bausteine

Aus genau zwei Strukturen lässt sich jede JSON-Datei zusammensetzen – beliebig ineinander verschachtelt:

Objekt  { }

Eine ungeordnete Sammlung von Schlüssel-Wert-Paaren, getrennt durch Komma. Schlüssel sind immer Strings in doppelten Anführungszeichen.
{"name": "Max", "alter": 20}

Array  [ ]

Eine geordnete Liste von Werten, getrennt durch Komma. Die Werte dürfen unterschiedliche Typen haben.
["rot", "grün", 3, true]

Erlaubte Datentypen (Werte)
  • string – Text in doppelten Anführungszeichen: "Hallo"
  • number – Ganz- oder Kommazahl, ohne Anführungszeichen: 42, 3.14, -7
  • boolean – Wahrheitswert, klein geschrieben: true / false
  • null – „kein Wert“, klein geschrieben: null
  • object – ein weiteres { … } (Verschachtelung)
  • array – eine weitere [ … ] (Verschachtelung)
📌
Merksatz: JSON = Text aus Objekten {} und Arrays []. Werte sind string · number · boolean · null · object · array – mehr gibt es nicht.
✍️ Syntax im Detail

Ein vollständiges Beispiel mit allen Datentypen und Verschachtelung:

{ "name": "Max Mustermann", // string "alter": 20, // number "azubi": true, // boolean "abschluss": null, // null = noch kein Wert "hobbys": ["Lesen", "Coden"], // array "adresse": { // verschachteltes object "stadt": "Berlin", "plz": "10115" } }

Hinweis: Die // Kommentare oben dienen nur der Erklärung – echtes JSON kennt keine Kommentare!

Die festen Regeln
Doppelte Quotes
Strings und Schlüssel stehen immer in doppelten Anführungszeichen "…" – niemals einfache '…'.
Schlüssel = String
Jeder Schlüssel in einem Objekt ist ein String: "alter": 20 – nicht alter: 20.
Komma trennt
Paare und Listenelemente werden mit Komma getrennt – aber kein Komma nach dem letzten Element.
Kleinschreibung
true, false, null werden klein geschrieben und stehen ohne Anführungszeichen.
Keine Kommentare
JSON erlaubt keine Kommentare (// oder /* */) und keine Funktionen.
Zahlen pur
Zahlen stehen ohne Anführungszeichen. "20" ist ein String, 20 eine Zahl – ein Unterschied!
🚫 Typische Fehler
⚠️
Diese drei Fehler kosten in der Prüfung Punkte:
1 · Quotes Einfache statt doppelter Anführungszeichen. {'name': 'Max'} ist ungültig → {"name": "Max"}.
2 · Trailing Comma Komma nach dem letzten Element. {"a": 1, "b": 2,} ist ungültig → das letzte Komma muss weg.
3 · Falsche Literale Python-Schreibweise im JSON: True/False/None statt true/false/null.
💡
Prüf-Trick: Gültiges JSON erkennst du an doppelten Quotes überall, keinem Komma am Ende und kleinem true/false/null.
🔬 JSON Zeichen für Zeichen

Hier wird JSON komplett seziert: jede Klammer, jedes Anführungszeichen, jeder Doppelpunkt und jedes Komma wird erklärt – dazu jeder Schlüssel und Wert samt Datentyp. Vier Stufen, gestaffelt von ganz einfach bis verschachtelt. Jede Stufe ist ein eigener Block: ein Klick öffnet ihn, ein Klick schließt ihn (standardmäßig zu).

🧭
So liest du die Aufschlüsselung: Oben steht das echte JSON, darunter wird es Zeile für Zeile erklärt (Zeichen → Bedeutung). Wichtig: Die Erklärungen stehen neben dem JSON – niemals darin, denn echtes JSON kennt keine Kommentare.

Das JSON

{

"kursId": 7,

"titel": "Python Grundlagen",
Schlüssel "titel"; Wert in Anführungszeichen = String; , → es geht weiter.
"aktiv": true,
Schlüssel "aktiv"; Wert true = boolean; , → es geht weiter.

"name": "Frau Berg",
Im inneren Objekt: Schlüssel "name", Wert = String; , → noch ein Paar im inneren Objekt.
"email": "berg@schule.de"
Schlüssel "email", Wert = String. Kein Komma – letztes Paar des inneren Objekts.

"themen": ["Variablen", "Schleifen"]

}

📌
Roter Faden: erst die Struktur ({ } und "…"), dann die Typen (String, Zahl, Boolean), dann Listen ([ ]) und zuletzt die Verschachtelung (Objekt im Objekt). Genau in dieser Reihenfolge sind die vier Stufen aufgebaut.
🐍 JSON ↔ Python: das json-Modul

Python bringt das Modul json von Haus aus mit. Es wandelt JSON-Text in Python-Objekte um (parsen) und Python-Objekte zurück in JSON-Text (serialisieren).

import json # --- String -> Python (parsen) --- text = '{"name": "Max", "alter": 20, "azubi": true}' daten = json.loads(text) # daten ist jetzt ein dict print(daten["name"]) # Max # --- Python -> String (serialisieren) --- person = {"name": "Anna", "azubi": True} text = json.dumps(person) # '{"name": "Anna", "azubi": true}' # --- Datei lesen / schreiben --- with open("daten.json", "w", encoding="utf-8") as f: json.dump(person, f) # Python -> Datei with open("daten.json", "r", encoding="utf-8") as f: daten = json.load(f) # Datei -> Python
Die vier Funktionen – das „s“ ist der Trick
json.loads()
load string: JSON-Text → Python-Objekt. (s = string)
json.dumps()
dump string: Python-Objekt → JSON-Text. (s = string)
json.load()
JSON aus einer Datei → Python-Objekt. (ohne s = Datei)
json.dump()
Python-Objekt → JSON in eine Datei. (ohne s = Datei)
📌
Merksatz: load = hereinladen (JSON→Python), dump = rauswerfen (Python→JSON). Das „s“ steht für string – mit „s“ geht es über einen Text, ohne „s“ über eine Datei.
So werden die Typen umgewandelt
JSONPython
object  {}dict
array  []list
stringstr
number (ganz)int
number (komma)float
true / falseTrue / False

⚠️
Stolperfalle: Beim Übersetzen tauschen sich true↔True, false↔False und null↔None. Schreibst du im JSON-Text aus Versehen True oder None, ist es kein gültiges JSON und json.loads wirft einen Fehler.
🔌 JSON in unserer Lernplattform

Diese Plattform ist ein gutes Live-Beispiel: Der Lernfortschritt der Karteikarten wird im Browser unter localStorage gespeichert. Aber localStorage kann nur Text (Strings) – kein Objekt. Deshalb wird genau die JSON-Brücke gebaut:

// Objekt -> JSON-Text -> speichern const fortschritt = { gemeistert: ["q1", "q5"], stand: 42 }; localStorage.setItem("fisi_json_v1", JSON.stringify(fortschritt)); // laden -> JSON-Text -> Objekt const roh = localStorage.getItem("fisi_json_v1"); // ein String const daten = JSON.parse(roh); // wieder ein Objekt

In JavaScript heißen die beiden Funktionen JSON.stringify() (Objekt → Text, wie Pythons dumps) und JSON.parse() (Text → Objekt, wie Pythons loads). Auch die Karteikarten- und Quiz-Daten in unseren Modulen sind nichts anderes als solche Objekt-/Array-Strukturen – also gelebtes JSON.

💡
Brücke zur Prüfung: Egal ob Python (json.dumps/loads) oder JavaScript (JSON.stringify/parse) – das Prinzip ist identisch: Objekt ⇄ Text. JSON ist die gemeinsame Sprache dazwischen.
🛠️ Praxis: dict in eine JSON-Datei schreiben

Drei aufeinander aufbauende Programmier-Aufgaben mit dem json-Modul. Erst selbst in IDLE / VS Code loesen, dann Hinweis oder Loesung anzeigen - ein Klick oeffnet die komplette Loesung samt Erklaerung.

Aufgabe 1 · dict erweitern und in Datei schreiben
Gegeben ist ein dict einwohner mit Stadt -> Einwohnerzahl.
  1. Ergaenze die Stadt Bonn mit 350000 Einwohnern.
  2. Schreibe das komplette dict mit json.dump in die Datei einwohnerdatei.json - gut lesbar eingerueckt (indent=10), encoding="utf-8".
Neuer Eintrag: Eine Zuweisung an einen noch nicht vorhandenen Schluessel legt ihn an: einwohner["Bonn"] = 350000.
Schreiben: Datei mit open(name, "w", encoding="utf-8") oeffnen, dann json.dump(obj, datei, indent=10) und am Ende datei.close().
Loesung (komplettes Programm):
import json einwohner = { "Berlin": 3669491, "Hamburg": 1847253, "Muenchen": 1484226, "Koeln": 1087863, "Frankfurt aM": 763380, "Stuttgart": 635911, "Duesseldorf": 621877, "Leipzig": 593145, "Dortmund": 588250, "Essen": 582760, "Bremen": 567559, "Dresden": 556780, "Hannover": 536925, "Nuernberg": 518370 } einwohner["Bonn"] = 350000 # neuen Schluessel-Wert ergaenzen do = open("einwohnerdatei.json", "w", encoding="utf-8") json.dump(einwohner, do, indent=10) # Python-dict -> JSON-Datei, Einzug 10 Zeichen do.close()
💡
Verstaendnis: json.dump (ohne "s") schreibt das Python-Objekt direkt in die geoeffnete Datei. Das dict wird dabei zu einem JSON-object; indent=10 rueckt jede Ebene um 10 Zeichen ein und macht die Datei gut lesbar.
⚠️
Typischer Fehler: json.dumps (mit "s") verwechseln - das gibt nur einen String zurueck und schreibt nichts in die Datei. Ebenso haeufig: die Datei nicht mit .close() schliessen, dann landen die Daten evtl. nicht vollstaendig auf der Platte.
📌
Merksatz: dump ohne "s" = Datei. Ablauf immer open -> dump -> close (oder ein with-Block, der automatisch schliesst).
Aufgabe 2 · verschachteltes dict (Liste als Wert)
Lege ein dict tn1 fuer einen Teilnehmer an mit den Schluesseln name (Text), Alter (Zahl) und hobbies.
  1. Der Wert von hobbies ist eine Liste aus mehreren Hobbys.
  2. Schreibe das dict mit json.dump in die Datei teilnehmer.json (indent=10, encoding="utf-8").
Liste als Wert: Ein dict-Wert darf selbst eine Liste sein: "hobbies": ["Fussball", "Schach"]. Die Liste steht in eckigen Klammern [ ], die einzelnen Hobbys als Strings mit Komma getrennt.
Schreiben: wie gehabt open(name, "w", encoding="utf-8"), dann json.dump(tn1, datei, indent=10), dann datei.close().
Loesung (komplettes Programm):
import json tn1 = {"name": "Otto Meier", "Alter": 32, "hobbies": ["Fussball", "Schach", "Chillen"]} do = open("teilnehmer.json", "w", encoding="utf-8") json.dump(tn1, do, indent=10) # dict mit Liste -> JSON-Datei do.close()
💡
Verstaendnis: Ein dict-Wert kann selbst eine Liste (oder ein weiteres dict) sein - so entsteht eine verschachtelte Struktur. Beim Schreiben wird die Python-Liste zu einem JSON-array [ ]; json.dump arbeitet sich rekursiv durch die ganze Verschachtelung.
⚠️
Typischer Fehler: Die Hobbys als einen String schreiben ("Fussball, Schach") statt als Liste - dann ist es keine Liste, sondern nur ein langer Text. Eckige Klammern [ ] nicht vergessen.
📌
Merksatz: dict-Werte duerfen selbst Listen oder dicts sein - beliebig tief verschachtelbar. Liste -> JSON-array [ ], dict -> JSON-object { }.
Aufgabe 3 · Liste von dicts (Tabelle)
Speichere zwei IT-Geraete - jedes mit den Schluesseln geraet, hersteller, preis, defekt.
  1. Lege das erste Geraet als dict an und schreibe es einzeln in geraet.json (indent=4).
  2. Lege ein zweites Geraet an, fasse beide in einer Liste geraete zusammen (= Tabelle) und schreibe die Liste in geraete.json (indent=4).
Tabelle: Eine Liste von dicts [geraet, geraet2] ist die typische Tabellen-Struktur - jedes dict ist eine Zeile, die gleichen Schluessel sind die Spalten.
Booleans: True/False in Python werden im JSON automatisch zu true/false. indent=4 = 4 Zeichen Einzug.
Loesung (komplettes Programm):
import json # 1. Dictionary fuer ein IT-Geraet geraet = {"geraet": "Notebook", "hersteller": "Dell", "preis": 899.99, "defekt": True} # 2. einzelnes Dictionary in Datei schreiben datei = open("geraet.json", "w", encoding="utf-8") json.dump(geraet, datei, indent=4) # dict -> JSON (indent = lesbar) datei.close() # 3. zweites Geraet, beide als Liste speichern geraet2 = {"geraet": "Drucker", "hersteller": "HP", "preis": 149.50, "defekt": False} geraete = [geraet, geraet2] # Liste von Dictionaries = Tabelle datei = open("geraete.json", "w", encoding="utf-8") json.dump(geraete, datei, indent=4) datei.close()
💡
Verstaendnis: Eine Liste von dicts ist die Standard-Form fuer eine Tabelle: jedes dict = ein Datensatz (Zeile), die gleichen Schluessel = Spalten. json.dump schreibt die ganze Liste als JSON-array von objects.
⚠️
Typischer Fehler: Statt einer Liste die dicts addieren (geraet + geraet2) - das geht nicht. Richtig ist die eckige Klammer: [geraet, geraet2]. Und: dieselbe Datei nach .close() fuer den zweiten Schreibvorgang neu oeffnen.
📌
Merksatz: Liste von dicts = Tabelle. json.dump schreibt beliebig verschachtelte Strukturen (dict / list / Zahl / String / bool) in einem Rutsch.

Aufgabe 5 · localStorage & JSON
Warum muss man ein Objekt vor dem Speichern in localStorage erst durch JSON.stringify() schicken – und beim Laden durch JSON.parse()?
  • localStorage kann nur Strings speichern, keine Objekte/Arrays.
  • JSON.stringify(obj) macht aus dem Objekt einen JSON-Text, der sich speichern lässt.
  • Beim Laden kommt wieder ein String zurück – JSON.parse(text) baut daraus das Objekt zurück.
  • Ohne parse hätte man nur einen Text und könnte nicht mit .eigenschaft zugreifen.
