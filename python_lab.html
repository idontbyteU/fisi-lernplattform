Schleifen verstehen

Eine Schleife wiederholt etwas. Hier siehst du Schritt für Schritt, welche Zahl gerade „dran“ ist und wie ein Zähler mitläuft – statt es dir nur vorzustellen.

🔁 for-Schleife mit range()
for i in range(start, stop, schritt) – der Marker wandert durch die Zahlen. Wichtig: stop ist nicht mehr dabei!
Wozu? Eine for-Schleife wiederholt etwas für eine bekannte Anzahl an Durchläufen. range() erzeugt dafür die Zahlenfolge. In echt: jede Zeile einer Datei durchgehen, alle Nutzer einer Liste abarbeiten oder 10 Würfe simulieren.
Drücke „Abspielen” oder geh „Schritt” für Schritt.
Durchlaufiprint(i)
📖 Code – Zeile für Zeile
for i in range(5):
Was macht das?
range(5) erzeugt die Zahlen 0, 1, 2, 3, 4 – fängt immer bei 0 an und hört vor der 5 auf. Die Variable i nimmt nacheinander jeden dieser Werte an: in Durchlauf 1 ist i = 0, in Durchlauf 2 ist i = 1 usw.
print(i)
Was macht das?
Gibt den aktuellen Wert von i aus. Diese Zeile läuft bei jedem Durchlauf einmal – weil sie eingerückt unter dem for steht. Einrückung (4 Leerzeichen) = „gehört zur Schleife”.
range(5) → 0, 1, 2, 3, 4  ·  range(1, 5) → 1, 2, 3, 4  ·  Regel: range hört immer VOR der letzten Zahl auf. Ohne Startzahl beginnt es bei 0.
🔂 while-Schleife: Summe bilden
Ein Akkumulator (summe) sammelt Werte, solange die Bedingung i <= n wahr ist. Du siehst, wie sich summe und i verändern.
Wozu? while wiederholt, solange eine Bedingung wahr ist – ideal, wenn du nicht vorher weißt, wie oft. In echt: auf gültige Eingabe warten, lesen bis Dateiende oder ein Spiel laufen lassen, bis der Spieler verliert.
summe startet bei 0, i bei 1.
Durchlaufisumme
📖 Code – Zeile für Zeile
summe = 0
Was macht das?
Legt die Variable summe mit dem Startwert 0 an. Wird einmal vor der Schleife ausgeführt. summe dient als Sammelbehälter – jeder Durchlauf addiert etwas dazu.
i = 1
Was macht das?
Legt den Zähler i mit dem Startwert 1 an. i wird in der Bedingung geprüft und am Ende jedes Durchlaufs um 1 erhöht.
while i <= n:
Was macht das?
Prüft vor jedem Durchlauf: Ist i noch kleiner oder gleich n? Ja → Schleifenkörper läuft. Nein → Schleife endet sofort. Die Prüfung passiert nach jedem Durchlauf erneut.
summe = summe + i
Was macht das?
Rechte Seite zuerst: summe + i wird berechnet. Dann wird das Ergebnis in summe gespeichert – der alte Wert ist weg. In Durchlauf 1: summe = 0 + 1 = 1. In Durchlauf 2: summe = 1 + 2 = 3. Usw.
i = i + 1
Was macht das?
Erhöht i um 1. Pflicht in jeder while-Schleife – ohne diese Zeile bleibt i immer 1, die Bedingung ist immer wahr, die Schleife hört nie auf (Endlosschleife!).

Listen bewegen

Listen sind geordnete Sammlungen. Jede Operation – anhängen, einfügen, löschen, sortieren – verschiebt die Kästchen sichtbar. Genau die Methoden aus deinen Schul-Dateien.

📋 Listen-Operationen live
Die Zahl unter jedem Kästchen ist der Index (Position) – er beginnt bei 0.
Wozu? Eine Liste hält viele Werte in fester Reihenfolge zusammen, die du jederzeit ändern kannst. In echt: Warenkorb, Highscore-Tabelle, Messwerte einer Stunde oder die Würfelergebnisse aus deinem Unterricht.
Liste bereit. Probiere die Operationen aus.
Merke: sort() und append() verändern die Liste direkt. sorted(liste) dagegen gibt eine neue sortierte Liste zurück und lässt das Original in Ruhe.
Algorithmus zum Zusehen

Bubblesort animiert

Bubblesort vergleicht immer zwei Nachbarn und tauscht sie, wenn sie falsch herum stehen. Große Werte „blubbern“ nach rechts. Gelb = wird verglichen, Rot = wird getauscht, Grün = fertig sortiert.

📊 Sortier-Visualizer
Wozu? Sortieren bringt Daten in Reihenfolge – Grundlage für Ranglisten, Suche und übersichtliche Ausgaben. Bubblesort zeigt das Prinzip Schritt für Schritt. In echt: Bestenliste nach Punkten, Namen alphabetisch, Produkte nach Preis – wobei man dafür liste.sort() nimmt.
Drücke „Sortieren“ – oder geh „Schritt“ für Schritt durch.
def bubblesort(liste): n = len(liste) for i in range(n - 1): for j in range(n - 1 - i): if liste[j] > liste[j + 1]: liste[j], liste[j + 1] = liste[j + 1], liste[j] return liste
Warum lernen? Bubblesort ist langsam, aber perfekt zum Verstehen, wie ein Algorithmus überhaupt arbeitet. Im Alltag nimmt man liste.sort() – das ist hochoptimiert.
Entscheidungen

if · elif · else

Das Programm wählt einen Weg, je nachdem ob eine Bedingung wahr ist. Verschiebe den Wert und sieh, welcher Zweig „anspringt“.

🔀 Notenrechner als Weggabelung
Python prüft von oben nach unten – der erste passende Zweig gewinnt, der Rest wird übersprungen.
Wozu? Mit if/elif/else reagiert ein Programm auf unterschiedliche Situationen. In echt: Login erlauben oder ablehnen, Versandkosten je nach Bestellwert, Ampel: rot/gelb/grün.
punkte = 82
Gib Punkte ein und drücke „Prüfen“.
Reihenfolge zählt: Stünde >= 50 ganz oben, würde auch eine 95 schon dort „hängenbleiben“. Darum von der schärfsten zur lockersten Bedingung ordnen.
Nachschlagen statt suchen

Dictionary (Schlüssel→Wert)

Ein Dictionary speichert Paare: zu jedem Schlüssel gehört ein Wert. Statt eine Liste zu durchsuchen, schlägst du direkt nach.

📒 Telefonbuch
Wozu? Ein Dictionary findet Werte sofort über ihren Schlüssel – kein Durchzählen nötig. In echt: Telefonbuch (Name→Nummer), Konfiguration (Einstellung→Wert), Lagerbestand (Artikel→Anzahl).
Trage Schlüssel + Wert ein und drücke „setzen“.
Liste vs. Dictionary: Liste = Reihenfolge per Zahl-Index (0,1,2…). Dictionary = Zugriff per sinnvollem Namen. Gleicher Schlüssel zweimal gesetzt → der alte Wert wird überschrieben.
Eigene Bausteine

Funktionen (def)

Eine Funktion ist eine kleine Maschine: Werte gehen rein (Parameter), drinnen passiert etwas, ein Ergebnis kommt raus (return).

⚙️ Wert rein → Verarbeitung → Wert raus
Wozu? Funktionen bündeln Code, den du benennst und immer wieder aufrufst – einmal schreiben, oft nutzen. In echt: Preis inkl. MwSt berechnen, E-Mail-Adresse prüfen, Wert formatieren – überall, wo sich Logik wiederholt.
Drücke „Aufrufen“, um die Funktion durchlaufen zu sehen.
return vs. print: print zeigt nur etwas an. return gibt einen Wert zurück, mit dem du weiterrechnen kannst.

String-Slicing

Mit text[start:stop] schneidest du einen Ausschnitt heraus. Jeder Buchstabe hat einen Index ab 0 – stop ist nicht mehr dabei.

✂️ Ausschnitt markieren
Wozu? Slicing holt gezielt Teile aus Text. In echt: Vorwahl aus einer Nummer, Dateiendung von einem Namen, Jahr aus einem Datum „2026-05-30“.
Ergebnis erscheint hier.
Tricks: text[:3] = von Anfang, text[3:] = bis Ende, text[-1] = letztes Zeichen.

Struktogramm & Pseudocode

Bevor man Code schreibt, plant man den Ablauf. In der AP1 geschieht das oft als Struktogramm (Bild) oder Pseudocode (halb Sprache, halb Code). Beides ist programmiersprachen-unabhängig – es zeigt die Logik, nicht die Syntax.

Was ist was?

Wozu? Struktogramm und Pseudocode beschreiben einen Ablauf, ohne sich um Python-Details (Doppelpunkte, Klammern) zu kümmern. So denkt man erst über die Lösung nach und übersetzt sie danach in echten Code. In echt: Aufgabe in der Prüfung planen, Ablauf im Team abstimmen, Logik dokumentieren.
  • Struktogramm (Nassi-Shneiderman): Kästchen, von oben nach unten gelesen.
  • Pseudocode: Schlüsselwörter wie WENN, SOLANGE, WIEDERHOLE – nah an Sprache, aber strukturiert.

Baustein 1 · Sequenz (nacheinander)

Anweisungen werden der Reihe nach abgearbeitet – einfach von oben nach unten.

lies Preis
lies Menge
gesamt = Preis × Menge
gib gesamt aus
Pseudocode
lies preis lies menge gesamt ← preis * menge gib gesamt aus
Python
preis = float(input()) menge = float(input()) gesamt = preis * menge print(gesamt)

Baustein 2 · Verzweigung (WENN … SONST)

Das Programm entscheidet sich für einen von zwei Wegen. Im Struktogramm steht die Bedingung oben, links der „wahr“-Zweig, rechts der „falsch“-Zweig.

alter < 18 ?
wahr (ja)gib „minderjährig“ aus
falsch (nein)gib „volljährig“ aus
Pseudocode
WENN alter < 18 DANN gib "minderjährig" aus SONST gib "volljährig" aus ENDE WENN
Python
if alter < 18: print("minderjährig") else: print("volljährig")

Baustein 3 · Schleife (SOLANGE / WIEDERHOLE)

Ein Teil wird wiederholt. Im Struktogramm umrahmt der Schleifenkopf den eingerückten Rumpf.

SOLANGE i ≤ 5
summe = summe + i
i = i + 1
gib summe aus
Pseudocode
summe ← 0 i ← 1 SOLANGE i <= 5 summe ← summe + i i ← i + 1 ENDE SOLANGE gib summe aus
Python
summe = 0 i = 1 while i <= 5: summe = summe + i i = i + 1 print(summe)

Pseudocode-Schlüsselwörter (IHK-Stil)

In der Prüfung tauchen meist diese Wörter auf. Sie sind nicht streng genormt – wichtig ist, dass die Struktur klar wird (Einrückung, Anfang/Ende).

  • WENN … DANN … SONST … ENDE WENN → Verzweigung (Python: if/else)
  • SOLANGE … ENDE SOLANGE → kopfgesteuerte Schleife (Python: while)
  • WIEDERHOLE … BIS → fußgesteuerte Schleife (mind. 1 Durchlauf)
  • FÜR i VON 1 BIS n → Zählschleife (Python: for i in range(1, n+1))
  • ← oder := → Zuweisung (Python: =)
Merke: = in Pseudocode/Mathe heißt oft „ist gleich?“ (Vergleich). Für „weise zu“ nutzt man dort ←. In Python ist = zuweisen und == vergleichen.

1 · Variablen & Datentypen

Eine Variable ist ein benannter Behälter für einen Wert. Python merkt sich den Typ automatisch.

  • int – ganze Zahl: alter = 20
  • float – Kommazahl: groesse = 1.80
  • str – Text (in Anführungszeichen): name = "Max"
  • bool – Wahrheitswert: aktiv = True
Im Code-Labor zeigt der Variablen-Inspektor jeden dieser Werte mit Typ an.

2 · Rechnen & Operatoren

Die Grundrechenarten plus drei, die oft in Prüfungen vorkommen:

  • ** Potenz · % Modulo (Rest) · // Ganzzahldivision
  • Beispiel: 10 % 3 ergibt 1 (Rest), 10 // 3 ergibt 3

3 · Ein- und Ausgabe

print(...) gibt aus, input(...) fragt ab. Wichtig: input() liefert immer Text – zum Rechnen erst mit int() oder float() umwandeln.

4 · Entscheidungen (if / elif / else)

Das Programm wählt einen Weg je nach Bedingung. Python prüft von oben nach unten – die erste passende Bedingung gewinnt.

5 · Schleifen (for & while)

Wiederholungen. for läuft über einen festen Bereich, while solange eine Bedingung wahr ist. → Sieh dir das im Tab Schleifen an.

9 · Dictionary & Strings
