Schleifen verstehen

Eine Schleife wiederholt etwas. Hier siehst du Schritt für Schritt, welche Zahl gerade „dran“ ist und wie ein Zähler mitläuft – statt es dir nur vorzustellen.

🔁 for-Schleife mit range()
for i in range(start, stop, schritt) – der Marker wandert durch die Zahlen. Wichtig: stop ist nicht mehr dabei!
Wozu? Eine for-Schleife wiederholt etwas für eine bekannte Anzahl an Durchläufen. range() erzeugt dafür die Zahlenfolge. In echt: jede Zeile einer Datei durchgehen, alle Nutzer einer Liste abarbeiten oder 10 Würfe simulieren.
Drücke „Abspielen” oder geh „Schritt” für Schritt.
Durchlaufiprint(i)
📖 Code – Zeile für Zeile
for i in range(5):
Was macht das?
range(5) erzeugt die Zahlen 0, 1, 2, 3, 4 – fängt immer bei 0 an und hört vor der 5 auf. Die Variable i nimmt nacheinander jeden dieser Werte an: in Durchlauf 1 ist i = 0, in Durchlauf 2 ist i = 1 usw.
print(i)
Was macht das?
Gibt den aktuellen Wert von i aus. Diese Zeile läuft bei jedem Durchlauf einmal – weil sie eingerückt unter dem for steht. Einrückung (4 Leerzeichen) = „gehört zur Schleife”.
range(5) → 0, 1, 2, 3, 4  ·  range(1, 5) → 1, 2, 3, 4  ·  Regel: range hört immer VOR der letzten Zahl auf. Ohne Startzahl beginnt es bei 0.
🔂 while-Schleife: Summe bilden
Ein Akkumulator (summe) sammelt Werte, solange die Bedingung i <= n wahr ist. Du siehst, wie sich summe und i verändern.
Wozu? while wiederholt, solange eine Bedingung wahr ist – ideal, wenn du nicht vorher weißt, wie oft. In echt: auf gültige Eingabe warten, lesen bis Dateiende oder ein Spiel laufen lassen, bis der Spieler verliert.
summe startet bei 0, i bei 1.
Durchlaufisumme
📖 Code – Zeile für Zeile
summe = 0
Was macht das?
Legt die Variable summe mit dem Startwert 0 an. Wird einmal vor der Schleife ausgeführt. summe dient als Sammelbehälter – jeder Durchlauf addiert etwas dazu.
i = 1
Was macht das?
Legt den Zähler i mit dem Startwert 1 an. i wird in der Bedingung geprüft und am Ende jedes Durchlaufs um 1 erhöht.
while i <= n:
Was macht das?
Prüft vor jedem Durchlauf: Ist i noch kleiner oder gleich n? Ja → Schleifenkörper läuft. Nein → Schleife endet sofort. Die Prüfung passiert nach jedem Durchlauf erneut.
summe = summe + i
Was macht das?
Rechte Seite zuerst: summe + i wird berechnet. Dann wird das Ergebnis in summe gespeichert – der alte Wert ist weg. In Durchlauf 1: summe = 0 + 1 = 1. In Durchlauf 2: summe = 1 + 2 = 3. Usw.
i = i + 1
Was macht das?
Erhöht i um 1. Pflicht in jeder while-Schleife – ohne diese Zeile bleibt i immer 1, die Bedingung ist immer wahr, die Schleife hört nie auf (Endlosschleife!).

Listen bewegen

Listen sind geordnete Sammlungen. Jede Operation – anhängen, einfügen, löschen, sortieren – verschiebt die Kästchen sichtbar. Genau die Methoden aus deinen Schul-Dateien.

📋 Listen-Operationen live
Die Zahl unter jedem Kästchen ist der Index (Position) – er beginnt bei 0.
Wozu? Eine Liste hält viele Werte in fester Reihenfolge zusammen, die du jederzeit ändern kannst. In echt: Warenkorb, Highscore-Tabelle, Messwerte einer Stunde oder die Würfelergebnisse aus deinem Unterricht.
Liste bereit. Probiere die Operationen aus.
Merke: sort() und append() verändern die Liste direkt. sorted(liste) dagegen gibt eine neue sortierte Liste zurück und lässt das Original in Ruhe.
Algorithmus zum Zusehen

Bubblesort animiert

Bubblesort vergleicht immer zwei Nachbarn und tauscht sie, wenn sie falsch herum stehen. Große Werte „blubbern“ nach rechts. Gelb = wird verglichen, Rot = wird getauscht, Grün = fertig sortiert.

📊 Sortier-Visualizer
Wozu? Sortieren bringt Daten in Reihenfolge – Grundlage für Ranglisten, Suche und übersichtliche Ausgaben. Bubblesort zeigt das Prinzip Schritt für Schritt. In echt: Bestenliste nach Punkten, Namen alphabetisch, Produkte nach Preis – wobei man dafür liste.sort() nimmt.
Drücke „Sortieren“ – oder geh „Schritt“ für Schritt durch.
def bubblesort(liste): n = len(liste) for i in range(n - 1): for j in range(n - 1 - i): if liste[j] > liste[j + 1]: liste[j], liste[j + 1] = liste[j + 1], liste[j] return liste
Warum lernen? Bubblesort ist langsam, aber perfekt zum Verstehen, wie ein Algorithmus überhaupt arbeitet. Im Alltag nimmt man liste.sort() – das ist hochoptimiert.

Dictionary (Schlüssel→Wert)

Ein Dictionary speichert Paare: zu jedem Schlüssel gehört ein Wert. Statt eine Liste zu durchsuchen, schlägst du direkt nach.

📒 Telefonbuch
Wozu? Ein Dictionary findet Werte sofort über ihren Schlüssel – kein Durchzählen nötig. In echt: Telefonbuch (Name→Nummer), Konfiguration (Einstellung→Wert), Lagerbestand (Artikel→Anzahl).
Trage Schlüssel + Wert ein und drücke „setzen“.
Liste vs. Dictionary: Liste = Reihenfolge per Zahl-Index (0,1,2…). Dictionary = Zugriff per sinnvollem Namen. Gleicher Schlüssel zweimal gesetzt → der alte Wert wird überschrieben.

Beispiel: 100.000 Zufallszahlen schreiben & eine bestimmte Zeile lesen Zum Lesen

Schreibt 100.000 Zufallszahlen (je 5 pro Zeile) in eine Datei und liest danach gezielt die 2389. Zeile wieder ein.

import random # Schreiben file = open("zufallszahlen.txt","w", encoding="utf-8") for i in range(1,100001): file.write(str(random.randint(0,999))) if i%5==0: file.write("\n") else: file.write("\t") file.close() # Lesen file = open("zufallszahlen.txt","r",encoding="utf-8") c=0 # Zeilenzähler for z in file: c=c+1 if c==2389: print(z) break file.close()
📖 Erklärung – Zeile für Zeile

— Schreiben —

  • open("…","w", encoding="utf-8"): öffnet/erstellt die Datei zum Schreiben. "w" = write, überschreibt eine vorhandene Datei. encoding legt die Zeichenkodierung fest.
  • range(1, 100001): zählt i von 1 bis 100000 (die 100001 ist ausgeschlossen).
  • file.write(str(random.randint(0,999))): schreibt eine Zufallszahl 0–999. str(...) ist Pflicht – write() nimmt nur Text, keine Zahlen.
  • if i%5==0: → "\n", sonst "\t": nach jeder 5. Zahl ein Zeilenumbruch, sonst ein Tab → 5 Zahlen pro Zeile, 20.000 Zeilen.
  • file.close(): schließt die Datei und schreibt den Puffer endgültig auf die Platte.

— Lesen —

  • open("…","r"): "r" = read, zum Lesen.
  • for z in file: geht die Datei Zeile für Zeile durch (z = eine ganze Zeile inkl. \n). Lädt nicht die ganze Datei in den Speicher – wichtig bei großen Dateien.
  • c zählt die Zeilen; bei c==2389 wird die 2389. Zeile ausgegeben und mit break abgebrochen.

— Typischer Fehler —

  • write() mit einer Zahl statt str(zahl) → TypeError.
  • "w" überschreibt; zum Anhängen "a" nehmen.
  • print(z) erzeugt eine Leerzeile extra, weil z schon ein \n enthält → print(z, end="") oder z.strip().

— Merksatz —

  • Besser als manuelles open/close: with open(...) as file: – schließt automatisch, auch bei Fehlern.
Workshop: Planetentabelle formatieren & als CSV speichern Geführte Übung

Acht Planeten-Datensätze sollen als saubere Tabelle ausgegeben und in planeten.csv gespeichert werden. Endziel ist genau diese Ausgabe:

Endziel – so soll die Ausgabe aussehen

| Name |Entf.Sonne(AE)|Radius(km)| Monde | |----------+--------------|----------|-------| | Merkur | 0.39 | 2440 | 0 | | Venus | 0.72 | 6052 | 0 | | Erde | 1.00 | 6378 | 1 | | Mars | 1.52 | 3397 | 2 | | Jupiter | 5.20 | 71493 | 79 | | Saturn | 9.54 | 60267 | 82 | | Uranus | 19.19 | 25559 | 27 | | Neptun | 30.07 | 24764 | 14 |

Deine Aufgaben (in planeten.py):

  1. Definiere für jeden Planeten eine Liste [Name, Entf.Sonne(AE), Radius(km), Monde] – z. B. merkur = ['Merkur', 0.39, 2440, 0].
  2. Lege eine Liste planetentabelle an und füge alle Planeten nacheinander mit planetentabelle.append(…) hinzu.
  3. Gib die Tabelle in einer Schleife aus: for p in planetentabelle: print(p[0], p[1], p[2], p[3]).
  4. Bring die Spalten mit einem f-String in Form – Breite, Ausrichtung und "|"-Spaltentrenner. (Start: print(f"{p[0]:7} {p[1]:12} {p[2]:8} {p[3]:3}"), dann anpassen.)
  5. Ergänze die Überschriftzeile über der Tabelle.
  6. Schreibe die planetentabelle in eine CSV-Datei planeten.csv.
✅ Komplette Lösung anzeigen (planeten.py)
import csv # 1. Planeten-Datensätze: [Name, Entf.Sonne(AE), Radius(km), Monde] merkur = ['Merkur', 0.39, 2440, 0] venus = ['Venus', 0.72, 6052, 0] erde = ['Erde', 1.0, 6378, 1] mars = ['Mars', 1.52, 3397, 2] jupi = ['Jupiter', 5.2, 71493, 79] saturn = ['Saturn', 9.54, 60267, 82] uranus = ['Uranus', 19.19, 25559, 27] neptun = ['Neptun', 30.07, 24764, 14] # 2. Alle Datensätze in eine Tabelle einsammeln planetentabelle = [] planetentabelle.append(merkur) planetentabelle.append(venus) planetentabelle.append(erde) planetentabelle.append(mars) planetentabelle.append(jupi) planetentabelle.append(saturn) planetentabelle.append(uranus) planetentabelle.append(neptun) # 5. Überschrift + Trennlinie kopf = f"| {'Name':<9}|{'Entf.Sonne(AE)':^14}|{'Radius(km)':^10}|{'Monde':^7}|" trenn = "|" + "-"*10 + "+" + "-"*14 + "|" + "-"*10 + "|" + "-"*7 + "|" print(kopf) print(trenn) # 3.+4. Tabelle formatiert ausgeben (f-String mit Breite/Ausrichtung) for p in planetentabelle: print(f"| {p[0]:<9}|{p[1]:>13.2f} |{p[2]:>9} |{p[3]:>5} |") # 6. In CSV-Datei schreiben with open("planeten.csv", "w", newline="", encoding="utf-8") as datei: writer = csv.writer(datei, delimiter=";") writer.writerow(["Name", "Entf.Sonne(AE)", "Radius(km)", "Monde"]) for p in planetentabelle: writer.writerow(p) print("\nGespeichert in planeten.csv")

f-String-Formatierung (Schritt 4)

  • Breite: die Zahl hinter dem Doppelpunkt reserviert Platz, z. B. {p[0]:9} = 9 Zeichen breit. So stehen die Spalten sauber untereinander.
  • Ausrichtung: < = linksbündig (für Text wie den Namen), > = rechtsbündig (für Zahlen), ^ = zentriert (hier für die Überschriften).
  • Nachkommastellen: :.2f erzwingt genau 2 Stellen → aus 1.0 wird 1.00. Kombiniert mit Breite: {p[1]:>13.2f} = rechtsbündig, 13 breit, 2 Nachkommastellen.
  • Die "|" zwischen den Platzhaltern sind einfache Trennzeichen, die als Text mitgedruckt werden.

CSV-Teil (Schritt 6)

  • Das csv-Modul übernimmt das korrekte Trennen/Escapen – man muss die Spalten nicht selbst per ";" zusammenbauen.
  • open(..., newline="", encoding="utf-8"): newline="" ist beim CSV-Schreiben Pflicht, sonst entstehen unter Windows leere Zeilen zwischen den Datensätzen.
  • writer.writerow(liste) schreibt eine ganze Liste als eine Zeile. Zuerst die Kopfzeile, dann in der Schleife jeden Planeten.
  • Ergebnis – planeten.csv:
    Name;Entf.Sonne(AE);Radius(km);Monde
    Merkur;0.39;2440;0 … usw.
