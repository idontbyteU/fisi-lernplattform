#05 Notensystem mit if/elif/else ✓ Erledigt
Schreibe ein Programm das eine Prozentzahl (0–100) einliest und die Schulnote ausgibt:
• 90–100 → "Sehr gut (1)"
• 75–89 → "Gut (2)"
• 60–74 → "Befriedigend (3)"
• 50–59 → "Ausreichend (4)"
• unter 50 → "Nicht bestanden (5)"

Prüfe außerdem ob die Eingabe überhaupt gültig ist (0–100).
💡 Hinweis
Struktur von if/elif/else:
if punkte >= 90:
  print("Sehr gut")
elif punkte >= 75:
  print("Gut")
else:
  print("...")

Python prüft von oben nach unten – sobald eine Bedingung zutrifft, werden die anderen übersprungen!
✓ Musterlösung
punkte = int(input("Punkte (0-100): ")) if punkte < 0 or punkte > 100: print("Ungültige Eingabe!") elif punkte >= 90: print("Sehr gut (1)") elif punkte >= 75: print("Gut (2)") elif punkte >= 60: print("Befriedigend (3)") elif punkte >= 50: print("Ausreichend (4)") else: print("Nicht bestanden (5)")
#06 FizzBuzz mit for-Schleife ✓ Erledigt
Klassische Programmieraufgabe! Gib die Zahlen von 1 bis 50 aus, aber:
• Bei Vielfachen von 3 → "Fizz"
• Bei Vielfachen von 5 → "Buzz"
• Bei Vielfachen von 3 und 5 → "FizzBuzz"
• Sonst → die Zahl selbst

Erwarteter Anfang: 1, 2, Fizz, 4, Buzz, Fizz, 7, ...
💡 Hinweis
for i in range(1, 51): geht von 1 bis 50 (51 ist nicht dabei!).

Modulo-Trick: i % 3 == 0 ist True wenn i durch 3 teilbar ist.

Wichtig: Der FizzBuzz-Fall (durch 3 UND 5) muss zuerst geprüft werden!
✓ Musterlösung
for i in range(1, 51): if i % 3 == 0 and i % 5 == 0: print("FizzBuzz") elif i % 3 == 0: print("Fizz") elif i % 5 == 0: print("Buzz") else: print(i)

#09 Schwimmbadkarte – Preisberechnung ✓ Erledigt
Schreibe ein Programm das den Eintrittspreis für ein Schwimmbad berechnet:
• Alter < 6 → kostenlos (0,00 €)
• Alter 6–12 → 2,00 €
• Alter 13–20 → 4,00 €
• Alter > 20 → 6,00 €

Gib den Preis mit einem f-String aus: "Der Preis ist: 4.00 €"

💡 Hinweis

#10 Schaltjahr berechnen ✓ Erledigt

Ein Jahr ist ein Schaltjahr wenn:
• Es durch 4 teilbar ist
• Außer es ist durch 100 teilbar
• Es sei denn es ist auch durch 400 teilbar

Teste mit: 1900 (kein SJ), 2000 (SJ), 2004 (SJ), 2006 (kein SJ)

j = int(input("Jahreszahl> ")) if j % 400 == 0: schaltjahr = True elif j % 100 == 0: schaltjahr = False elif j % 4 == 0: schaltjahr = True else: schaltjahr = False print("Prüfung abgeschlossen.") if schaltjahr: print(f"{j} ist ein Schaltjahr") else: print(f"{j} ist kein Schaltjahr")

Debugging lernen

Fehler lesen &
selbst lösen

Python-Fehlermeldungen sagen dir genau was falsch ist – wenn du weißt wie man sie liest.

Die goldene Debugging-Regel: Lies die letzte Zeile der Fehlermeldung zuerst. Dort steht der Fehlertyp und die Ursache. Dann schau auf die Zeilennummer (line X) um zu wissen wo der Fehler ist.
Häufige Fehlertypen
🔴 SyntaxError – Tippfehler im Code
Der Code kann gar nicht erst starten. Häufig: fehlende Doppelpunkte, falsche Klammern, vergessene Anführungszeichen.
x = 10 if x > 5 # ← Doppelpunkt fehlt! print("groß")
File "test.py", line 2 if x > 5 ^ SyntaxError: expected ':'
Fix: Füge den Doppelpunkt nach der if-Bedingung hinzu: if x > 5:

Merksatz: Nach if, for, while, def, else, elif kommt immer ein :
🔴 TypeError – Falsche Datentypen
Du versuchst eine Operation mit inkompatiblen Typen, z.B. String + Integer.
alter = input("Alter: ") print("In 10 Jahren: " + alter + 10)
TypeError: can only concatenate str (not "int") to str
Fix: print("In 10 Jahren:", int(alter) + 10)

Ursache: input() gibt immer einen String zurück. Du kannst keinen String und eine Zahl mit + verbinden. Entweder umwandeln mit int() oder str() benutzen.
🔴 NameError – Variable nicht definiert
Du benutzt eine Variable die es (noch) nicht gibt – oft ein Tippfehler im Namen.
name = "Max" print(Name) # ← Großschreibung!
NameError: name 'Name' is not defined
Fix: print(name) – Python unterscheidet Groß- und Kleinschreibung!

Merksatz: Variablennamen in Python immer komplett kleinschreiben (Konvention). name ≠ Name ≠ NAME
🔴 IndexError – Liste außerhalb der Grenzen
Du versuchst auf ein Listenelement zuzugreifen das nicht existiert.
farben = ["rot", "blau", "grün"] print(farben[3]) # ← gibt es nicht! (0, 1, 2)
IndexError: list index out of range
Fix: Listen starten bei Index 0. Eine Liste mit 3 Elementen hat die Indizes 0, 1, 2.
farben[2] → "grün" (letztes Element)
farben[-1] → letztes Element (Trick!)

Check: len(farben) gibt die Länge zurück. Gültiger Bereich: 0 bis len-1.
🔴 IndentationError – Falsche Einrückung
Python benutzt Einrückung (4 Leerzeichen oder 1 Tab) um Blöcke zu definieren. Fehler hier = Programm startet nicht.
for i in range(3): print(i) # ← muss eingerückt sein!
IndentationError: expected an indented block after 'for' statement
Fix: Alles innerhalb einer Schleife/if/Funktion muss eingerückt sein:
for i in range(3):
    print(i) ← 4 Leerzeichen!

Tipp: Benutze immer 4 Leerzeichen (oder Tab, aber nicht mischen!). IDLE und VS Code machen das automatisch.
Debugging-Strategie
1. Fehlermeldung lesen – Letzte Zeile = Fehlertyp + Ursache. Zeilennummer zeigt wo.
2. print() als Detektiv – Füge print("hier bin ich", variable) ein um zu sehen was passiert.
3. Minimal-Test – Schreibe den verdächtigen Code separat mit festen Werten.
4. Google den Fehlertyp – "Python TypeError can only concatenate" liefert sofort die Antwort.
5. Debugger benutzen – In IDLE: Debug → Debugger. In VS Code: F5 und Breakpoints setzen.
