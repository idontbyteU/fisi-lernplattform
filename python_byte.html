#06 FizzBuzz mit for-Schleife ✓ Erledigt
Gib die Zahlen von 1 bis 50 aus, aber:
• Vielfache von 3 → "Fizz"
• Vielfache von 5 → "Buzz"
• Vielfache von 3 und 5 → "FizzBuzz"
• Sonst → die Zahl selbst

Anfang: 1, 2, Fizz, 4, Buzz, Fizz, 7, ...
💡 Hinweis
for i in range(1, 51): geht 1 bis 50.

i % 3 == 0 ist True wenn i durch 3 teilbar.

Der FizzBuzz-Fall (3 UND 5) muss zuerst geprüft werden!
✓ Musterlösung

#18 AP1: Schwellenwert prüfen (> vs. >=) ✓ Erledigt
Typische AP1-Falle. Ein Server gilt als überlastet, wenn die Auslastung über 80 % liegt – genau 80 % zählt noch nicht als überlastet.

Lies auslastung ein und gib "ÜBERLASTET" oder "OK" aus. Teste mit 80 und 81.
💡 Hinweis
Achte auf > vs. >=:
• "über 80" = auslastung > 80 (80 selbst ist NICHT dabei)
• "ab 80" / "mindestens 80" wäre >= 80

Die obere Grenze gehört bei "bis unter" immer zur besseren/anderen Kategorie.
✓ Musterlösung
auslastung = int(input("Auslastung in %: ")) if auslastung > 80: print("ÜBERLASTET") else: print("OK") # Test: # 80 -> OK (80 ist NICHT > 80) # 81 -> ÜBERLASTET (81 ist > 80)
Debugging lernen

Fehler lesen &
selbst lösen

Python-Fehlermeldungen sagen dir genau was falsch ist – wenn du weißt wie man sie liest.

Goldene Regel: Lies die letzte Zeile der Fehlermeldung zuerst – dort steht Fehlertyp und Ursache. Dann die Zeilennummer (line X) für den Ort.
Häufige Fehlertypen
🔴 SyntaxError – Tippfehler im Code
Der Code startet gar nicht erst. Häufig: fehlende Doppelpunkte, Klammern, Anführungszeichen.
x = 10 if x > 5 # ← Doppelpunkt fehlt! print("groß")
File "test.py", line 2 if x > 5 ^ SyntaxError: expected ':'
Fix: Doppelpunkt anhängen: if x > 5:

Merksatz: Nach if, for, while, def, else, elif kommt immer ein :
🔴 TypeError – Falsche Datentypen
Operation mit inkompatiblen Typen, z. B. String + Integer.
alter = input("Alter: ") print("In 10 Jahren: " + alter + 10)
TypeError: can only concatenate str (not "int") to str
Fix: print("In 10 Jahren:", int(alter) + 10)

Ursache: input() gibt einen String zurück. String + Zahl geht nicht – mit int() umwandeln.
🔴 NameError – Variable nicht definiert
Du benutzt eine Variable, die es nicht gibt – oft ein Tippfehler.
name = "Max" print(Name) # ← Großschreibung!
NameError: name 'Name' is not defined
Fix: print(name) – Python unterscheidet Groß-/Kleinschreibung! name ≠ Name
🔴 IndexError – Liste außerhalb der Grenzen
Zugriff auf ein Listenelement, das nicht existiert.
farben = ["rot", "blau", "grün"] print(farben[3]) # ← gibt es nicht! (0,1,2)
IndexError: list index out of range
Fix: Index startet bei 0. 3 Elemente → Indizes 0,1,2. farben[2] = "grün", farben[-1] = letztes Element.
🔴 IndentationError – Falsche Einrückung
Python nutzt Einrückung (4 Leerzeichen) für Blöcke.
for i in range(3): print(i) # ← muss eingerückt sein!
IndentationError: expected an indented block after 'for' statement
Fix: Inhalt von Schleife/if/Funktion einrücken – immer 4 Leerzeichen. Nicht Tabs und Leerzeichen mischen!
Debugging-Strategie
1. Fehlermeldung lesen – letzte Zeile = Typ + Ursache.
2. print() als Detektiv – print("hier", variable) einbauen.
3. Minimal-Test – verdächtigen Code separat mit festen Werten testen.
4. Fehlertyp googeln – liefert meist sofort die Antwort.
5. Debugger – IDLE: Debug → Debugger. VS Code: F5 + Breakpoints.
