🔗 Angriffsphasen (Kill Chain)

  • Initial Access – Phishing, Schwachstellen, gestohlene Zugangsdaten
  • Execution / Foothold – erster stabiler Code läuft
  • Persistence – Autostart, Tasks, Dienste
  • Privilege Escalation – Normal → Admin → Domain Admin
  • Lateral Movement – Ausbreitung im Netzwerk
  • Discovery / Collection – Erkundung, Daten sammeln
  • Exfiltration – Daten nach außen abziehen
  • Impact – Ransomware, Sabotage, DDoS, Erpressung

🦠 Malware-Arten

  • Virus – hängt sich an Dateien, verbreitet sich per Ausführung
  • Wurm – selbstständig über Netzwerkschwachstellen
  • Trojaner – Tarnung als legitime Software
  • Ransomware – Verschlüsselung + Lösegeld
  • Spyware / Keylogger – Daten & Tastatur ausspähen
  • Rootkit – Kernel-Level, extrem schwer erkennbar
  • Botnet / C2 – ferngesteuerte Systeme (DDoS, Spam)
  • APT – langfristig, zielgerichtet, oft staatlich gesponsert

🎭 Social Engineering

  • Phishing – gefälschte E-Mails / Webseiten
  • Spear-Phishing – personalisiert / gezielt gegen Einzelne
  • CEO-Fraud / Whaling – Zahlungsprozesse manipulieren
  • Vishing / Smishing / QRishing – Tel., SMS, QR-Code
  • Pretexting – erfundene glaubwürdige Geschichte
  • Baiting / USB Drop – präparierte Datenträger auslegen
  • MFA-Fatigue – Anfragenflut bis versehentlich bestätigt
  • Credential Stuffing / Spraying – automatisierte Passwortangriffe
⚠ Warnsignal: Zeitdruck · „sofort" · „vertraulich" · „Chef-Anweisung" · „letzte Chance"

🌐 Cyberphysisch & Infrastruktur

  • MitM – zwischen Kommunikationspartner sitzen, mitlesen
  • Sniffing – passives Mitschneiden (kritisch: unverschlüsselt!)
  • ARP-Spoofing – LAN-Traffic über Angreifer umleiten
  • DNS-Spoofing / Pharming – auf falsche IPs leiten
  • Rogue Wi-Fi / Evil Twin – gefälschter WLAN-Accesspoint
  • Replay-Angriff – aufgezeichnete Auth erneut absenden
  • DoS / DDoS – Dienste durch Überlastung lahmlegen

🌍 Web- & DB-Angriffe

  • XSS – bösartiges JavaScript in Webseiten einschleusen
  • CSRF – ungewollte Aktionen im Nutzerkontext auslösen
  • SSRF – Server ruft interne/externe Ziele auf
  • Broken Access Control – Zugriff auf fremde Daten
  • Session Hijacking – gestohlene Cookies / Tokens
  • SQL Injection – DB-Abfragen über Eingabefelder manipulieren
  • DB-Fehlkonfiguration – offene Ports, Standardkonten
Grundregel: Eingaben niemals ungeprüft vertrauen · Berechtigungen serverseitig prüfen

🤖 KI-Bedrohungen

  • Data Poisoning – Trainingsdaten vergiften
  • Adversarial Examples – KI täuschende Eingaben
  • Prompt Injection – LLM-Systemfilter umgehen
  • Unsichere Tool-Nutzung – KI + API ohne Kontrolle
  • Deepfakes – Audio/Video-Fälschung für CEO-Fraud
  • KI-Phishing – skalierbar, personalisiert, mehrsprachig
  • Datenlecks in Trainingsdaten – Modell gibt Infos preis
