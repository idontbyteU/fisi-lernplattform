🧭 Was ist BPMN?

BPMN (englisch Business Process Model and Notation, frueher „Business Process Modelling Notation“) ist eine standardisierte, grafische Sprache, um Geschaeftsprozesse darzustellen. Ein BPMN-Diagramm liest sich wie ein Satz von links nach rechts: Es beginnt mit einem Start-Ereignis, folgt dem Ablauf ueber Aktivitaeten und Verzweigungen (Gateways) und endet mit einem End-Ereignis.

🎯
Pruefungsrelevanz: BPMN ist seit 2025 Teil der AP1 (an die Stelle von u. a. SQL, RAID, Struktogrammen und PAP). Gefragt ist vor allem die IHK-Notation: einen beschriebenen Prozess auf Papier zeichnen bzw. ein Lueckendiagramm vervollstaendigen.

📖 Glossar - die wichtigsten Begriffe

Die Kernbausteine, die in (fast) jeder Aufgabe vorkommen. Die gezeichneten Symbole dazu findest du im Tab Symbole.

BPMN Notation

Business Process Model and Notation - die grafische Sprache selbst. Aktuell in Version 2.0. Beschreibt einen Prozess pro Diagramm.

Ereignis Event

Etwas, das eintritt und den Ablauf ausloest, unterbricht oder beendet - gezeichnet als Kreis. Drei Grundtypen: Start (duenner Kreis), Zwischen (Doppelkreis), Ende (dicker Kreis).

Message Event Nachricht

Ereignis-Typ mit Briefumschlag: eine Nachricht trifft ein oder wird gesendet. Kann als Start-, Zwischen- oder Endevent vorkommen (z. B. eine Bestellung als Message-Startevent).

Timer Event Uhr

Ereignis-Typ mit Uhr: der Ablauf wartet auf Zeit - einen Zeitpunkt oder eine Dauer (z. B. 'nach 5 Arbeitstagen'). Meist als Zwischenereignis.

Exception Event Fehler

Ereignis-Typ mit Blitz: ein Fehler/eine Ausnahme tritt ein und unterbricht den normalen Ablauf.

Aktivitaet Task

Eine Taetigkeit/Arbeit, die im Prozess verrichtet wird - abgerundetes Rechteck. Beschriftung moeglichst als Verb + Objekt (z. B. „Antrag pruefen“).

Teilprozess Sub-Process

Eine Aktivitaet, die selbst aus mehreren Schritten besteht und zusammengeklappt dargestellt wird - abgerundetes Rechteck mit kleinem Plus (+) am unteren Rand.

Gateway XOR exklusiv

Verzweigung, bei der genau ein Pfad genommen wird (Entweder-oder) - Raute, oft mit X. Typische Frage am Gateway, z. B. „Genehmigt?“ mit den Ausgaengen ja/nein.

Gateway OR inklusiv

Verzweigung, bei der mindestens ein Pfad gewaehlt wird - von einem bis zu allen - Raute mit Kreis (O).

Gateway AND parallel

Verzweigung, bei der alle Pfade gleichzeitig ausgefuehrt werden (Und) - Raute mit Plus (+). Wird mit einem zweiten AND-Gateway wieder zusammengefuehrt (synchronisiert).

Sequenzfluss Reihenfolge

Durchgezogener Pfeil - verbindet Flussobjekte innerhalb desselben Pools und legt die Reihenfolge fest.

Nachrichtenfluss Message

Gestrichelter Pfeil mit offener Pfeilspitze - zeigt den Austausch einer Nachricht zwischen zwei Pools (Beteiligten), nicht innerhalb eines Pools.

Datenobjekt Artefakt

Ein Artefakt (Blatt mit umgeknickter Ecke), das die im Prozess verwendeten Daten/Dokumente abbildet (z. B. 'Antrag', 'Lieferschein'). Verbindet sich ueber eine Assoziation, nicht ueber den Sequenzfluss.

Pool / Lane Swimlane

Pool = ein Beteiligter (z. B. Unternehmen, Kunde). Lane = eine Rolle/Abteilung innerhalb eines Pools (z. B. Verkauf, Lager). Sie zeigen die Zustaendigkeit „wer macht was“.

📌
Merke fuer die IHK-Notation: innerhalb eines Pools durchgezogene Pfeile (Sequenz), zwischen Pools gestrichelte Pfeile (Nachricht). Gateways immer paarweise denken: was per XOR/AND aufgeteilt wird, mit demselben Gateway-Typ wieder zusammenfuehren.
🧩 Symbol-Lernhilfe

Das IHK-Kernset - genau die Symbole, die du fuer die AP1 zeichnen koennen musst. Die Form traegt die Bedeutung: der besondere Teil eines Symbols ist hier farbig hervorgehoben.

Regeln der Notation
  • Jeder Prozess beginnt und endet mit einem Ereignis (Kreis) - nie mit einer Aktivitaet oder einem Gateway.
  • Ein Gateway trifft keine Entscheidung, es bildet die Verzweigung ab. Die Bedingung steht am Pfad, die Logik passiert in den Aktivitaeten davor.
  • Der Sequenzfluss bleibt im Pool. Ueber Pool-Grenzen hinweg verbindet nur der Nachrichtenfluss (gestrichelt).
  • XOR (exklusiv) = genau ein Pfad, OR (inklusiv) = mindestens einer (eins bis alle), AND (parallel) = alle gleichzeitig.
Ereignisse - der Rahmen (wann im Ablauf?)

Der Rahmen sagt, WO im Ablauf: duenn = Start -> doppelt = Zwischen -> dick = Ende.

Startevent Start

Duenner, einfacher Kreis. Markiert den Beginn des Prozesses, von dem der Ablauf nach rechts losgeht.

Intermediate Event Zwischen

Doppelkreis. Steht mitten im Ablauf (zwischen Start und Ende) - z. B. um zu warten oder eine Nachricht zu melden.

Endevent Ende

Dicker, fetter Kreis. Markiert das Ende eines Pfades. Ein Prozess darf mehrere Enden haben.

Ereignis-Typen - das Innensymbol (was loest aus?)

Das Innensymbol sagt, WAS: jeder Typ kann als Start-, Zwischen- oder Endevent vorkommen - einfach mit dem passenden Rahmen kombinieren.

Message Event Nachricht

Kreis mit Briefumschlag. Eine Nachricht trifft ein oder wird gesendet (z. B. eine Bestellung als Message-Startevent).

Timer Event Uhr

Kreis mit Uhr. Der Ablauf wartet auf Zeit - einen Zeitpunkt oder eine Dauer, z. B. 'nach 5 Arbeitstagen'.

Exception Event Fehler

Kreis mit Blitz. Ein Fehler/eine Ausnahme tritt ein und unterbricht den normalen Ablauf.

Aktivitaeten (abgerundete Rechtecke)

Task / Aufgabe Aktivitaet

Abgerundetes Rechteck mit Beschriftung als Verb + Objekt. Eine konkrete Taetigkeit, die im Prozess verrichtet wird.

Teilprozess Sub-Process

Abgerundetes Rechteck mit Plus (+) am unteren Rand. Eine Aktivitaet, die selbst aus mehreren Schritten besteht und zugeklappt dargestellt wird.

Gateways (Rauten)

XOR / Exklusiv entweder-oder

Raute mit X. Es wird genau ein Pfad genommen (entweder/oder). Steht meist an einer Ja/Nein-Frage wie 'Genehmigt?'.

OR / Inklusiv eins bis alle

Raute mit Kreis (O). Es wird mindestens ein Pfad gewaehlt - von einem bis zu allen.

AND / Parallel und

Raute mit Plus (+). Alle Pfade laufen gleichzeitig (und). Wird mit einem zweiten AND-Gateway wieder zusammengefuehrt (synchronisiert).

Verbindungen (Pfeile)

Sequenzfluss Reihenfolge

Durchgezogener Pfeil mit voller Spitze. Verbindet Flussobjekte und legt die Reihenfolge fest - nur innerhalb eines Pools.

Nachrichtenfluss Message

Gestrichelter Pfeil mit offener Spitze (Kreis am Anfang). Verbindet Prozesse ueber Poolgrenzen hinweg - nie innerhalb eines Pools.

Artefakte (Daten)

Datenobjekt Daten

Blatt mit umgeknickter Ecke. Ein Artefakt, das die im Prozess verwendeten Daten/Dokumente abbildet (z. B. 'Antrag', 'Lieferschein').

Organisation (Zustaendigkeit)

Pool mit Lanes Swimlane

Pool = ein Beteiligter (definiert die Prozessgrenzen). Lanes = Rollen/Abteilungen darin. Sie zeigen, wer was macht.

📌
Pruefungstrick: Bei Ereignissen erst den Rahmen lesen (duenn/doppelt/dick = Start/Zwischen/Ende), dann das Innensymbol (Umschlag/Uhr/Blitz). Bei Gateways die Raute, dann das Zeichen (X/Kreis/+). Pfeile zuletzt: durchgezogen im Pool, gestrichelt zwischen Pools.
